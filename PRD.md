## 1. 背景與目標

### 1.1 背景

Cofit 是一個連結營養師（Coach）與使用者（Client）的健康管理平台，提供 Web 與 App 雙端服務。目前登入 / 註冊入口為使用者與 Coach 進入系統的必要路徑，直接影響新用戶轉換率與既有用戶留存率。

### 1.2 目標

| 目標 | 衡量指標 |
|------|----------|
| 提升新用戶完成註冊率 | 目標：開始填寫 → 完成驗證 ≥ 70% |
| 降低因忘記帳密造成的客服量 | 目標：相關客服 ticket 減少 30% |
| 統一帳號管理入口（基本資料、履歷、付款） | 設定頁功能完整上線 |

---

## 2. 使用者角色

| 角色 | 說明 |
|------|------|
| **新用戶** | 第一次使用 Cofit，需完成 Email 註冊與驗證 |
| **既有用戶** | 已有帳號，日常登入使用 |
| **Coach（營養師）** | 需填寫履歷資料、管理付款資訊與版本升級 |

---

## 3. 功能範圍

```
登入模組
├── 登入
├── 登入錯誤（密碼格式）
└── 登入失敗（帳密不符）

註冊模組
├── 填寫 Email + 密碼
├── 驗證信寄出
├── 驗證連結過期
└── 註冊完成

├── 手機號碼輸入
├── OTP 驗證

└── 帳號顯示

忘記密碼
├── 輸入 Email
├── 重設連結寄出
├── 設定新密碼
└── 重設成功

設定頁（側邊欄）
├── 基本資料
├── 履歷資料
├── 版本升級
├── 付款資訊
├── 通知中心
└── 文件下載
```

## 4. 功能需求

### 4.1 登入

#### 畫面元素
- Cofit Logo
- Email 輸入框（placeholder：`Email註冊帳號`）
- 密碼輸入框（placeholder：`輸入密碼 (至少為8位英文字母或數字)`）、顯示/隱藏密碼切換
- 記住我 Checkbox
- 忘記密碼連結
- 登入按鈕（Primary）
- 忘記帳號連結
- 切換至「註冊」Tab

#### 互動邏輯

| 情境 | 條件 | 行為 |
|------|------|------|
| 密碼格式錯誤 | 密碼長度 < 8 位 | 顯示「登入_密碼格式錯誤」畫面 |
| 帳密不符 | 格式正確但帳密不存在 | 顯示「登入失敗」錯誤提示 |
| 登入成功 | 帳密正確 | 跳轉至首頁 / Dashboard |

#### 錯誤訊息規格

- 密碼不足 8 位：顯示 inline 錯誤，輸入框邊框變紅（`#e53935`）
- 帳密不符：卡片頂部顯示紅底錯誤 Alert Banner

---

### 4.2 註冊

#### 畫面元素
- Email 輸入框
- 密碼輸入框（含顯示/隱藏切換）
- 確認密碼輸入框（含顯示/隱藏切換）
- 服務條款同意文字（含可點擊連結）
- 註冊按鈕（Primary）

#### 驗證規則（前端）

| 欄位 | 規則 | 錯誤訊息 |
|------|------|----------|
| Email | 符合 `^[^\s@]+@[^\s@]+\.[^\s@]+$` | 請輸入正確的 Email 格式 |
| 密碼 | 長度 ≥ 8、含英文字母且含數字 | 密碼強度不足，至少需 8 位英文字母及數字 |
| 確認密碼 | 與密碼欄位完全相符 | 兩次輸入的密碼不一致 |
| Email（後端回傳） | 未被其他帳號使用 | 此 Email 已被使用，請改用其他信箱或直接登入 |

#### 驗證信流程

1. 填寫完成後送出 → 後端寄送驗證信
2. 顯示「驗證信已寄出」確認頁，告知用戶至信箱點擊連結
3. 若連結已過期 → 顯示「連結過期」畫面，提供重新發送選項
4. 點擊有效連結 → 跳轉「註冊完成」頁，引導至登入或首頁

---

### 4.3 忘記帳號

#### 流程

```
輸入手機號碼 → 傳送 OTP → 輸入 OTP → 顯示對應帳號 Email
```

#### 規格

| 步驟 | 說明 |
|------|------|
| 手機號碼輸入 | 台灣格式（09xxxxxxxx），送出後寄送簡訊 OTP |
| OTP 輸入 | 6 位數字，逾時時間建議 5 分鐘 |
| OTP 錯誤 | inline 錯誤提示，允許重新輸入；超過錯誤次數（建議 5 次）鎖定 |
| 帳號顯示 | 顯示對應 Email，部分遮蔽（如 `u***@cofit.me`） |

---

### 4.4 忘記密碼

#### 流程

```
輸入 Email → 寄送重設連結 → 點擊連結 → 設定新密碼 → 重設成功
```

#### 規格

| 步驟 | 說明 |
|------|------|
| Email 輸入 | 驗證格式；若 Email 不存在仍顯示「已寄出」（避免帳號枚舉攻擊） |
| 重設連結 | 有效期限 30 分鐘，單次使用 |
| 新密碼設定 | 同註冊密碼規則（≥ 8 位、英文+數字） |
| 確認密碼 | 兩次輸入需一致，不符顯示 inline 錯誤 |
| 重設成功 | 顯示成功頁面，引導至登入 |

### 4.5 設定頁（側邊欄）

#### 佈局規格

| 區域 | 規格 |
|------|------|
| Topbar | 高度 56px，左側 Logo，右側使用者頭像 + 姓名 + 下拉箭頭 |
| Sidebar | 寬度 240px，白底，6 個 nav item |
| 主內容區 | 彈性寬度，背景 `#f5f5f5`，內部 Card 白底 |

#### Sidebar Nav Items

| Icon | 項目 | 說明 |
|------|------|------|
| 👤 人像 | 基本資料 | 個人資料編輯 |
| 📄 文件 | 履歷資料 | 專業證照與認證 |
| ⭐ 星形 | 版本升級 | 會員方案升級 |
| 💳 信用卡 | 付款資訊 | 付款方式管理 |
| 🔔 鈴鐺 | 通知中心 | 通知偏好設定 |
| ⬇ 下載 | 文件下載 | 相關文件下載 |

Active 狀態：左側 3px teal（`#2D7B7B`）指示條 + 文字與 Icon 變色

#### 基本資料欄位

| 欄位 | 類型 | 必填 | 說明 |
|------|------|------|------|
| 頭像 | 圖片上傳 | 否 | 圓形顯示，支援上傳照片 |
| 姓氏 | Text | 是 | |
| 名字 | Text | 是 | |
| App 內暱稱 | Text | 是 | 顯示於 App 內 |
| 註冊帳號 | 顯示 | — | 不可編輯 |
| 使用者 ID | 顯示 | — | 不可編輯 |
| 備用信箱 | Email | 否 | |
| 手機號碼 | Tel | 否 | |
| 密碼 | 連結 | — | 點擊導至更改密碼流程 |

專屬連線碼、系統 ID、會員方案顯示於頭像區右側。

#### 履歷資料

採 Tab 切換：認證資訊 / 最高學歷 / 經歷 / 自我介紹 / 企業認證

**認證資訊 Tab 欄位：**

| 區塊 | 項目 |
|------|------|
| 使用者身分 | 營養師（國考合格勾選，顯示認證狀態 chip） |
| 專業證照 | CDE、CKD、體重管理師、保健食品初/研、HACCP |
| 體適能相關 | ACSM、NASM、NSCA、ACE、其他 |
| 長照相關 | 其他 |
| 專業認證概覽 | 已認證 N / 待上傳 N / 審核中 N |

## 5. 非功能需求

| 類別 | 規格 |
|------|------|
| **效能** | 登入頁首屏渲染 < 2 秒（LCP） |
| **安全性** | 密碼傳輸全程 HTTPS；密碼不明文儲存（bcrypt）；忘記帳號防枚舉 |
| **相容性** | 支援 Chrome、Safari、Edge 最新 2 個版本；iOS 15+ / Android 10+ |
| **無障礙** | 輸入框需有 label；錯誤訊息需有 aria-live；色彩對比度 WCAG AA |
| **國際化** | 目前僅支援繁體中文；語系架構預留 i18n 擴充 |

## 7. 錯誤狀態彙整

| 流程 | 錯誤情境 | 顯示方式 |
|------|----------|----------|
| 登入 | 密碼格式錯誤 | 全頁錯誤畫面 |
| 登入 | 帳密不符 | Alert Banner（卡片頂部） |
| 註冊 | Email 格式錯誤 | Inline（輸入框下方紅字） |
| 註冊 | 密碼強度不足 | Inline |
| 註冊 | 密碼不一致 | Inline |
| 註冊 | Email 已使用 | Inline |
| 驗證信 | 連結過期 | 全頁提示 + 重新發送按鈕 |
| OTP | 驗證碼錯誤 | Inline（輸入框下方） |
| 忘記密碼 | 密碼不一致 | Inline |
